# Markdownify — Desktop App

A double-click app that converts documents into clean Markdown text. No Python,
no command line — staff pick files, click Convert, get `.md` files next to the
originals. Two engines, switched with a segmented control (a pill toggle):

- **⚡ Quick** (MarkItDown) — fast, instant. Word, Excel, PowerPoint, text PDFs.
- **🔬 Deep** (Docling) — slower, ML-powered. Understands layout, tables, and
  scanned pages (OCR). Runs **fully offline** from models bundled in the app.

## Files here

| File | What it is |
| --- | --- |
| `markitdown_gui.py` | The whole app (two-tab window + both engines). |
| `requirements.txt` | Pinned dependencies (markitdown + docling). |
| `fetch_models.py` | Downloads Docling's models into `assets/docling_models`. |
| `make_icon.py` | Draws the app icon from the FD logo. |
| `build_macos.sh` | One-command macOS build. Run on a Mac. |
| `build_windows.bat` | One-command Windows build. Run on Windows. |
| `assets/` | Logo + bundled `docling_models/` (~1.2 GB, offline models). |
| `Markdownify.spec` | PyInstaller config (auto-generated by the build). |
| `dist/Markdownify.app` | The **finished macOS app** (~2.3 GB). |

## Try the macOS app now

1. Open `dist/` in Finder.
2. **Right-click** `Markdownify.app` → **Open** (right-click the first time only —
   it's unsigned, so a normal double-click is blocked by Gatekeeper).
   First launch is slow (~1 min) while macOS security-scans the 2.3 GB app.
3. Pick a mode (**⚡ Quick** or **🔬 Deep**) on the switch → **Choose files…** →
   the `.md` appears next to the original. Deep's first conversion takes ~10–25 s
   (loading models).

## Build the Windows version

A Windows build **cannot** be produced on a Mac — it must run on Windows.

1. Copy this `app/` folder to a Windows PC.
2. Install Python from python.org (**3.10–3.13**; 3.14 is too new for some deps).
3. Double-click `build_windows.bat`. It installs both engines, downloads the
   Docling models, and packages everything (produces a ~2.3 GB `dist\Markdownify\`).

## Rebuild the macOS version

```
./build_macos.sh
```
(Needs `brew install python-tk@3.14` first if tkinter is missing.)

## Data handling (regulated environment)

- **Everything runs locally.** No file and no converted text is uploaded.
- **Deep is fully offline** — its models are bundled; it never contacts the
  internet (the build sets `HF_HUB_OFFLINE`). Safe for locked-down networks.
- No AI image-description / audio-transcription features are enabled — no
  outbound calls of any kind.
- Loop in IT Risk (Thevan) and Compliance (John) before real client/employee docs.

## Sizes & limits (the trade-off you accepted)

- **~2.3 GB per app** — the Deep engine bundles PyTorch + Docling models. This is
  the cost of a single fully-offline app that also does OCR/tables.
- Quick still can't read chart-only/scanned PDFs — that's exactly what Deep is for.
- Deep is CPU-heavy: ~10–25 s per PDF on a typical laptop.
